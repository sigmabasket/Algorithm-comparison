Название задачи: Поиск подстроки

Описание

Этот проект предназначен для сравнительного анализа эффективности алгоритмов поиска подстроки:
 • Naive Search
 • Knuth-Morris-Pratt (KMP)
 • Boyer-Moore
 • Rabin-Karp
 • Apostolico-Crochemore
 • Aho_Corasick


Производится автоматическое тестирование по:
 • времени выполнения (со статистикой и доверительными интервалами),
 • потреблению оперативной памяти (RAM),
 • разным сценариям (лучший, худший и случайный случай),
 • разным размерам входных данных (от 1 КБ до 16 МБ).

Структура проекта
 • algorithms.py — реализация алгоритмов поиска.
 • data_generator.py — генерация тестовых данных.
 • time_measurer.py — замер времени выполнения с доверительными интервалами.
 • memory_measurer.py — замер потребления RAM.
 • benchmark.py — главный модуль запуска тестов.
 • time_results.json, memory_results.json — результаты тестирования
(создаются автоматически при запуске тестирования).
 • results.csv — все результаты моего тестирования.
 • benchmark.log — лог-файл ошибок и выполнения.
 • requirements.txt - необходимые модули для запуска.
 • readme.txt — описание проекта.
 • plot_time_results.py - построение графиков производительности для алгоритмов.
 • plot_memory_results.py - построение графиков RAM для алгоритмов.
 • substring_report.docx - финальный отчёт по тестированию алгоритмов.
 • test_all.py - тестирование основных методов.

Установка зависимостей

Требуются следующие сторонние библиотеки (можно установить через pip):
 • tqdm
 • matplotlib
 • numpy

Желательно использовать Python 3.8+

Запуск итогового тестирования:
python benchmark.py
 • Так как обрабатываются и сохраняются только ещё не протестированные размеры,
 то перед тестированием нужно очистить уже заполненные файлы json.

Программа:
 • адаптивно подбирает количество запусков (n_runs) в зависимости от размера входа,

 • прогревает каждый алгоритм 5 раз,
 • сохраняет результаты после каждого завершённого блока (алгоритм + случай),
 • логирует ошибки и этапы выполнения.

Формат выходных файлов

time_results.json:
{
  "naive": {
    "best": [
      {
        "size": 1024,
        "time": 0.00011,
        "delta": 0.00003
      },
      ...
    ]
  }
}

• size — длина строки.
 • time — среднее время выполнения.
 • delta — доверительный интервал (95%).

memory_results.json

Аналогичный формат, но для памяти (в КБ).

Особенности
 • Используется t-таблица для расчёта доверительных интервалов.
 • Прогревочные запуски исключаются из статистики.
 • Тестируются 3 типа случаев для каждого алгоритма.
 • Обрабатываются и сохраняются только ещё не протестированные размеры.

Вся информация касающаяся непосредственно алгоритмов и результатов их тестирования
представлена в отчёте (substring_report.docx).
